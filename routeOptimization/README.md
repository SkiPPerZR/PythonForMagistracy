## Вариант 1. Оптимизация маршрутов

Проект реализует систему поиска оптимальных маршрутов между городами по трём критериям: длина, время и стоимость. Для каждого запроса строятся три маршрута, оптимальных по каждому критерию отдельно, а также один компромиссный маршрут с учётом заданных приоритетов. 

## Формат входных данных

Программа читает файл `input.txt` в корне проекта. Файл состоит из трёх секций. 

### Секция [CITIES]

Каждая строка описывает город:

`ID: Название_города`

- `ID` - натуральное число.
- `Название_города` - строка, может содержать пробелы. 

### Секция [ROADS]

Каждая строка описывает дорогу:

`ID1 - ID2: длина(км), время(мин), стоимость(руб)`

- Дороги двусторонние.
- Все параметры - целые положительные числа. 

### Секция [REQUESTS]

Каждая строка описывает запрос:

`Город_отправления -> Город_назначения | (Д,В,С)`

- Названия городов совпадают с названиями из секции `[CITIES]`.
- В скобках задаются приоритеты критериев:
  - `Д` - длина
  - `В` - время
  - `С` - стоимость
- Порядок символов означает важность: первый - самый важный, третий - наименее важный (например, `(Д,С,В)`).

## Формат выходных данных

Результат записывается в файл `output.txt` в корне проекта. Для каждого запроса выводятся четыре строки:-

ДЛИНА: Город1 -> ... -> ГородN | Д=..., В=..., С=...
ВРЕМЯ: Город1 -> ... -> ГородN | Д=..., В=..., С=...
СТОИМОСТЬ: Город1 -> ... -> ГородN | Д=..., В=..., С=...
КОМПРОМИСС: Город1 -> ... -> ГородN | Д=..., В=..., С=...

- Строки `ДЛИНА`, `ВРЕМЯ`, `СТОИМОСТЬ` соответствуют маршрутам, оптимальным по длине, времени и стоимости соответственно.
- В блоке `Д=..., В=..., С=...` всегда выводятся суммарные значения всех трёх параметров для найденного маршрута.
- Строка `КОМПРОМИСС` описывает маршрут, выбранный по комбинированному критерию, учитывающему приоритеты.

Если маршрут между двумя городами не существует, выводится строка вида:

ДЛИНА: ПУТЬ НЕ НАЙДЕН

Аналогично для других критериев.

## Алгоритмы и структуры данных

- Карта дорог представлена как неориентированный граф со списком смежности: вершины - города, рёбра - дороги с тройкой весов (длина, время, стоимость).
- Для поиска маршрутов используется алгоритм Дейкстры с очередью с приоритетом (`heapq`) отдельно для каждого критерия: минимизация длины, времени и стоимости.
- Компромиссный маршрут строится с помощью модифицированного Дейкстры, где вес ребра считается как линейная комбинация трёх параметров с коэффициентами, зависящими от порядка приоритетов (например, 3, 2, 1).

## Требования

- Python 3.11 или выше.

## Установка и запуск

1. Склонируйте репозиторий:

```bash
git clone https://github.com/SkiPPerZR/PythonForMagistracy/tree/main/routeOptimization
cd routeOptimization
```


2. Убедитесь, что в корне проекта находится файл `input.txt` в описанном формате.

3. Запустите программу:

```bash
python main.py
```


4. После выполнения в этой же директории появится файл `output.txt` с результатами.

## Организация кода

- `main.py` - точка входа, парсинг `input.txt`, запуск алгоритмов, формирование `output.txt`.
- Внутри файла:
- `Graph` - структура графа и реализация Дейкстры для режимов `Д`, `В`, `С` и комбинированного критерия.
- Функции парсинга входного файла и обработки запросов.
- Функции форматирования строк вывода в соответствии с требованиями задания.
